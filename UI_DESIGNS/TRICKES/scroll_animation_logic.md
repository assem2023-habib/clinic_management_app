# توثيق حركة ظهور العناصر عند التمرير (Scroll Animations)

تم استخدام تقنية متطورة لدمج اكتشاف الرؤية (Visibility Detection) مع الحركات البرمجية (Programmatic Animations) لضمان ظهور العناصر بشكل جمالي فقط عندما تصبح مرئية للمستخدم أثناء التمرير.

---

## 1. التقنيات المستخدمة
تعتمد هذه الحركة على مكتبتين أساسيتين:
1. **`visibility_detector`**: لمراقبة موقع العنصر على الشاشة ومعرفة متى يدخل منطقة الرؤية.
2. **`flutter_animate`**: لتنفيذ الحركات البصرية (التلاشي، الانزلاق، تغيير الحجم).

---

## 2. آلية العمل (Step-by-Step)

### أ. مراقبة الرؤية (The Observer)
يتم تغليف كل بطاقة شهري (`MonthlyPayrollCard`) بـ `VisibilityDetector`. هذا الـ Widget يقوم بإطلاق حدث `onVisibilityChanged` باستمرار أثناء التمرير.

```dart
VisibilityDetector(
  key: Key('monthly_card_${widget.month}'),
  onVisibilityChanged: (info) {
    // إذا ظهر أكثر من 20% من العنصر ولم يسبق له الظهور
    if (info.visibleFraction > 0.2 && !_isVisible) {
      setState(() {
        _isVisible = true; // تفعيل حالة الظهور
      });
    }
  },
  child: ...
)
```

### ب. ربط الحالة بالحركة (The Controller)
بدلاً من تشغيل الحركة فور بناء العنصر (والتي قد تحدث وهو خارج الشاشة)، نستخدم المتغير `_isVisible` كمتحكم في هدف الحركة (`target`).

```dart
.animate(target: _isVisible ? 1 : 0) // إذا كان مرئياً، انتقل للقيمة 1 (تشغيل الحركة)
.fadeIn(duration: 400.ms)
.scale(begin: const Offset(0.9, 0.9))
.slideY(begin: 0.1)
```

---

## 3. تفاصيل "تركيب العنصر" (Component Assembly)

ما يجعل الحركة تبدو كأنها "تتركب" هو استخدام تأخيرات زمنية (Delays) مختلفة لكل جزء داخل البطاقة الواحدة:

1. **إطار البطاقة الخارجي**: يظهر أولاً مع تأثير `scale` خفيف ليعطي إيحاء بالعمق.
2. **العنوان والشهر**: ينزلقان من اليمين بعد `100.ms`.
3. **أرقام المستحقات والسلف**: تظهر بتتابع سريع من الأسفل.
4. **صافي الرواتب (البطاقة الداخلية)**: تظهر متأخرة بـ `600.ms` مع تأثير `shimmer` (لمعان) لجذب الانتباه للرقم النهائي.
5. **زر التفاصيل**: يظهر كآخر قطعة في "التركيبة" بعد `800.ms`.

---

## 4. لماذا هذه الطريقة؟ (The Benefit)
- **الأداء**: لا يتم استهلاك موارد المعالج في تشغيل حركات لعناصر غير مرئية.
- **تجربة المستخدم (UX)**: يشعر المستخدم أن التطبيق يتفاعل مع حركته، حيث "تولد" العناصر وتترتب أمام عينيه مباشرة أثناء نزوله في القائمة.
- **الاحترافية**: تعطي انطباعاً بأن التطبيق مصمم بعناية فائقة للتفاصيل (High-end feel).
